# Workflow: Generate / refresh the web dashboard

**Objective:** Keep the static web dashboard at `https://monkalways.github.io/offer-optics/` in sync with the upstream data pipeline. The web dashboard replaces the "at-a-glance" role of the Google Sheets dashboard (which stays current in parallel for ad-hoc tabular queries).

## When to run

- **Weekly** while the 2025-26 cycles are still receiving submissions (through ~end of May 2026)
- After any change to `config/justin_profile.json`, `config/requirements.yaml`, or the placement model in `tools/analyze_program.py`
- After any refresh of the upstream Google Sheets (same trigger as the Sheets dashboard refresh)
- **Immediately** after any real-world event that should be reflected in the action items (e.g. Justin confirms Harvard MEDScience enrollment, HYRS decision arrives, a new grade locks in)

## Architecture recap (why there are three hand-written files and one generated file)

```
docs/
├── index.html   ← hand-written scaffold (rare edits)
├── styles.css   ← hand-written custom layer (rare edits)
├── app.js       ← hand-written renderers + charts (rare edits)
└── data.json    ← GENERATED by tools/build_webdash.py (refresh every pipeline run)
```

A normal data refresh only touches `data.json`. The visual layer (HTML/CSS/JS) is committed and static, so we only re-touch it if we deliberately want to change the design.

## Tools

- `tools/build_webdash.py` — reads SQLite + profile + analysis JSONs + requirements YAML, writes `docs/data.json`. **Idempotent**; safe to re-run. Also emits a terse summary of the output shape.

## Data sources consumed

| Source | Used for |
|---|---|
| `.tmp/applications.sqlite` `placement` table | Per-program verdicts, percentiles, reasoning text reference |
| `.tmp/applications.sqlite` `requirements` table | Deadlines, supp app types, official URLs, fees |
| `.tmp/applications.sqlite` `applications` table | Histogram bins, YoY trends, decision timeline |
| `.tmp/applications.sqlite` `cycles` table | Data quality section |
| `.tmp/analysis/{program_key}.json` | Full reasoning text per program (same content that ends up in placement.reasoning) |
| `config/justin_profile.json` | Profile strip, top-6 card, summer_2026_plans for action items, `ec_strengths_by_target` for accordion |
| `config/requirements.yaml` | Source URLs and curator confidence levels |

All paths are gitignored except `config/`, so refreshing does not touch anything version-controlled except the newly-generated `docs/data.json`.

## Full refresh procedure

```bash
# 1. Optionally refresh upstream data (skip if only local profile/config changed)
venv/Scripts/python.exe tools/fetch_sheet.py --cycle all

# 2. Normalize + build SQLite (cheap; always run)
venv/Scripts/python.exe tools/normalize_data.py
venv/Scripts/python.exe tools/build_sqlite.py
venv/Scripts/python.exe tools/load_requirements.py

# 3. Re-compute Justin's placement
venv/Scripts/python.exe tools/analyze_program.py

# 4. Refresh both deliverables
venv/Scripts/python.exe tools/build_dashboard.py     # Sheets (unchanged)
venv/Scripts/python.exe tools/build_webdash.py       # Web -> docs/data.json

# 5. Local smoke test before pushing
venv/Scripts/python.exe -m http.server 8000 --directory docs
# Open http://localhost:8000/ and eyeball the verdict tiles, distribution chart,
# and action items. Stop the server (Ctrl-C) when satisfied.

# 6. Commit and push (deploys to GitHub Pages)
git add docs/data.json
git commit -m "Refresh dashboard data $(date +%Y-%m-%d)"
git push origin main
```

The only file that should show up as modified on a normal refresh is `docs/data.json`. If the HTML/CSS/JS files also changed, that means you (or I) deliberately edited the visual layer — double-check before pushing.

## One-time GitHub Pages setup

Done once, never again:

1. Go to https://github.com/monkalways/offer-optics/settings/pages
2. **Source**: "Deploy from a branch"
3. **Branch**: `main` · **Folder**: `/docs`
4. Click Save.
5. After the first push that includes `docs/`, GitHub Pages builds and serves the site at `https://monkalways.github.io/offer-optics/` within ~1 minute.

## Editing the visual layer

If you want to change colors, add a section, or adjust the layout:

- `docs/index.html` — section structure, Tailwind config, CDN script tags
- `docs/styles.css` — custom CSS variables, reveal animation, pull quote, verdict badges, accordion, print styles
- `docs/app.js` — data fetching, rendering, Chart.js charts, accordion/sort interactions

After editing any of these, run the local smoke test (`python -m http.server 8000 --directory docs`) before committing. The browser console is your friend — any fetch error, JS exception, or Chart.js misconfig will show up there.

**Rule of thumb**: if you're editing HTML/CSS/JS, you probably DO want to commit those files. If you're only refreshing data, you only commit `docs/data.json`.

## Rebuild from scratch (disaster recovery)

If `docs/` gets corrupted or deleted:

1. Re-run `tools/build_webdash.py` — regenerates `docs/data.json` from SQLite.
2. The HTML/CSS/JS files are in git history — `git checkout main -- docs/index.html docs/styles.css docs/app.js docs/README.md` restores them.

## Known quirks

- **Browsers block `fetch()` from `file://`** — you must serve via HTTP to view locally. The page shows a clear red error banner with the fix if you open it wrong.
- **Chart.js parent needs explicit height** — `styles.css` sets `.chart-box` heights; don't remove them or the canvases collapse to 0px.
- **Tailwind config load order** — the inline `window.tailwind = { config: ... }` must stay before `<script src="https://cdn.tailwindcss.com"></script>`. Reversing the order silently breaks custom colors.
- **The Justin marker plugin is passed inline** to the distribution chart only (via `plugins: [justinMarkerPlugin]`). Don't `Chart.register()` it globally or it bleeds onto the YoY line chart.
- **Cycle dates are shifted +1 year** from the `config/requirements.yaml` template (which uses the 2025-26 cycle as a reference). When real 2026-27 dates are published, update the template and the shift becomes redundant.
